今晚不想干活, 翻译一下julia 1.8更新内容, 权当娱乐了, 原文请戳我

const可以用在可变复合类型的字段上了

julia

mutable struct T
    x::Int
    const y::Float64
end

julia
挺有用的, 可以把常量定义在复合类型中, 编译器应该也会进行优化

可以在调用时使用@inline宏了

之前的@inline只能在方法定义的时候添加, 每次调用都会以inline的方式. 现在可以在某个具体的调用时再用inline了:@inline f(x).

平时不太涉及性能优化, 还没用过@inline, 捂脸!

全局变量可以加类型标注了

julia

x::Int = 0
# > 0

x = "string"
# ERROR: MethodError: Cannot `convert` an object of type String to an object of type Int64

julia
很重要, 写代码更不容易出bug, 编译器也更能优化了.

为@threads添加了新的调度器

可我还没用过@threads!

早期的julia对@threads是用的静态调度, iteration => task => thread 之间的分派关系是预先定义好的: 每个task分配给一个thread. 现在的:dynamic可以动态地调度task, 按照每个迭代的工作量一样的前提假设去调度.

举了个栗子:

julia

# 一个loop, 指定的时间后终止
julia> function busywait(seconds)
            tstart = time_ns()
            while (time_ns() - tstart) / 1e9 < seconds
            end
        end

julia> @time begin
            Threads.@spawn busywait(5)
            Threads.@threads :static for i in 1:Threads.nthreads()
                busywait(1)
            end
        end
# 6.003001 seconds (16.33 k allocations: 899.255 KiB, 0.25% compilation time)

julia> @time begin
            Threads.@spawn busywait(5)
            Threads.@threads :dynamic for i in 1:Threads.nthreads()
                busywait(1)
            end
        end
# 2.012056 seconds (16.05 k allocations: 883.919 KiB, 0.66% compilation time)

julia
